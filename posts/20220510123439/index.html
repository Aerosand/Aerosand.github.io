OpenFOAM 2022-05-10 OpenFOAM文件结构

发表于2022-05-10|更新于2022-06-17|OpenFOAM
|字数总计:1.2k|阅读时长:4分钟|评论数:

OpenFOAM有以下几个文件夹。

1. OpenFOAM 文件夹

OpenFOAM本体文件都在 OpenFOAM - 9 里面，

  • etc: The OpenFOAM environment and global settings, needed for everything
  • tutorials: Usage examples
  • applications: Source code of solvers and utilities
  • src: Source code of libraries, used by the applications
  • wmake: Command and instructions for compilation of applications and libraries. Followed by a look at Allwmake and the Make directories
  • build: Intermediate compilation files, for applications and libraries
  • platforms: Final binaries for applications and libraries.
  • bin: Executable bash scripts
  • doc: Doxygen source files and coding style instructions
  • modules: Additional modules

2. User文件夹

我们使用和本体相同的文件结构来开发自己的项目源代码。这当然不是必须的，但是非常推荐这么做，因为项目的文件结构始终保持了统一。

  • applications will be used for our own developed solvers and utilities
  • src will be used for our own developed libraries
  • run will be used for our cases, including running the original tutorials

2.2. 库文件(src)

src文件夹下包含了众多的库（库中又包含了各式各样的类）。同时也包含了Make文件夹，Make文件夹中的 files 文件是为了告诉编译器怎么去编译库以及类，还有编译结果的存放位置。而同级的 options 文件则是告诉编译器包含文件的路径，以及 link 哪些库。

When we copy one class of a library to another location we have to remember that it most likely uses other parts of the original library, and we thus have to add the original library to the Make/options file.

3. 小结

所以用户开发一般包含3个部分：

  • applications：求解器
    • 主代码.C
    • 头文件.H
    • Make文件
  • src：头文件及库文件
    • 库本身
      • 不同的小类
      • Make文件
  • run：case案例
    • myCase
      • 初始条件文件夹
      • constant文件夹
        • polyMesh：生成的网格文件
        • transportProperties等物理参数文件
      • system文件夹
        • blockMesh文件
        • controlDict文件
        • fvSchemes文件
        • fvSolution文件
