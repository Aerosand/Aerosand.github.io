《C++ How to Program》 07 指针 pointers

1. 介绍

通过这一篇的讨论，我们会知道什么时候使用指针，以及如何去使用指针。在前面的讨论中，我们了解并使用了引用和按引用传递。

指针也使用按引用传递，可以被用来创建和操作动态数据结构，这些数据结构可以增长和收缩，例如链表、队列、堆栈和树。(Pointers also enables pass-by-reference and can be used to create and manipulate dynamic data structures that can grow and shrink, such as linked lists, queues, stacks and trees.) 这一篇先介绍一些关于指针的基础知识，后续文章会深入讨论指针强大的功能。

我们还会展示 built-in arrays（继承自C语言） 和 pointers 的区别。从上一篇我们可以看到C++提供了更好用更强大的标准库类 array 和 vector 。同样的，C++也提供两种字符串的写法，一种是前面讨论过的 string 类对象的用法（C++），还有一种是继承自C语言的 pointer-based string （C string）。

本次讨论将深入 C string，来帮助了解 pointers 和 built=in-arrays。 C string 广泛使用在过去的C和C++代码中。当然，我们现在更应该使用 array对象、vector对象，而不是 built-in-arrays，使用string对象而不是C string。

经过本次的讨论，我们会在后面深入看到使用了指针和引用的多态处理的巨大用处（polymorphic processing associated with object-oriented programming with references and pointers）。

2. 指针变量的声明和初始化

指针变量的值就是指向的内存地址。一般来说，变量包含一个特定值，而指针就是指向这个值的内存地址。

A variable name directly references a value, and a pointer indirectly references a value.

2.1. 声明和初始化

  • 指针的声明

指针和其他的变量一样，必须在使用之前声明。

int* countPtr, count; 表示，int*类型的指针countPrt指向变量count。

我们通常在变量的名字后加上 Ptr 来命名对应的指针。

  • 指针的初始化

指针要么声明指向一个变量，要么声明成一个空指针(nullptr)。

对声明的指针都进行初始化，以避免内存相关的错误。

3. 指针的按引用传递

在C++中一共有三种方式给函数传递参数：

  • pass-by-value 按值传递
  • pass-by-reference with a reference argument 通过参数引用的按引用传递
  • pass-by-reference with a pointer argument 通过指针变量的按引用传递

指针，就像一般引用一样，也可以在调用中实现对变量的修改，以及传递庞大的数据从而避免冗余的数据复制过程。

Passing a variable by reference with a pointer does not actually pass anything by reference, a pointer to that variable is passed by value and is copied into the function’s corresponding pointer parameter. The called function can then access that variable in the caller simply by dereferencing the pointer, thus accomplishing pass-by-reference.

4. 内置数组Built-in Arrays

4.1. 声明和初始化

c语言格式的数组用法

type arrayName[arraySize];

对数组直接初始化，超出数量报错，不足数量默认为0.

int n[5]{50, 20, 30, 40, 10};

或者根据提供的元素来决定数组大小。

int n[]{50, 20, 30, 40, 10};

4.2. 传递给函数

The value of a built-in array’s name is implicitly convertible to the address of the built-in array’s first element.

数组的名字隐式的被转换为首个元素的地址，也就是

arrayName → &arrayName[0]

注意： 传递数组给函数直接输入数组的名字就行了，需要注意的是，函数内对数组的修改会直接对数组本身生效。一般来说，程序都使用const限定符来限制数组数据的修改，除非确实需要。

我们可以直接在 function header 中声明 built-in array parameters

例如：

int sumElements(const int values[], const size_t numberOfElements)

不同于 array objects, 这里的 built-in array 并不知道自己的大小，所以函数参数需要接受 array 以及 arraySize。

也可以写成

int sumElements(const int* values, const size_t numberOfElements)

编译器并不区分 built-in 数组和指针，第一种形式会被编译器转成第二种形式一样处理，也就是说第二句中的 value 是指针，指针也就是数组变量中第一个元素的地址，这是一样的。

虽然上述两种方式是可以互换的，但是在传递数组的时候，还是使用第一种方式表示它是 built-in array。

4.3. 数组排序

和 array 对象的排序函数类似，built-in array 也可以使用排序函数。

begin 和 end 函数的使用需要包含函数头<iterator>，接收 built-in array 作为函数参数，返回指针，represent ranges of elements 用来表示元素的范围。

4.4. built-in array 的局限性和必要性

现代化的C++编程种，应该尽量的使用 array 对象 或者 vector 对象。

内置数组有很多的局限性：

  1. 不能直接使用关系符和等于号直接比较大小，而是需要元素之间逐个比较。
  2. 之间不能互相赋值，此时数组名字只是一个常指针。
  3. 内置数组不知道自己的大小，函数在处理它们时，分别需要数组的名字和数组的大小。
  4. 不能提供边界检查，必须自己检查是否满足边界要求。

但是在很多场合，内置数组还是需要的，比如在处理命令行参数的时候。

5. 常指针的使用

在什么时候使用 const 来限定函数参数呢，我们参考 least privilege。也就是说，always give a function enough access to the data in its parameters to accomplish its specified task, but no more.

Built-in arrays are always passed by reference and could easily be changed in the called function.

const 和 指针 的搭配一共有四种情况，分别讨论如下。

5.1. Nonconstant Pointer to Nonconstant Data

Highest access is granted. 最高权限。

The data can be modified through the dereferenced pointer, and the pointer can be modified to point to other data.

5.2. Nonconstant Pointer to Constant Data

A pointer that can be modified to point to any data of the appropriate type, but the data to which it points cannot be modified through that pointer.

指针可以修改为指向其他数据，但是这些数据不能通过指针来被修改。

例如：

const int* countPtr: countPtr is a non-constant pointer to integer constant

注意：这种写法下，哪个是 constant，哪个是non-constant

5.3. Constant Pointer to Nonconstant Data

A pointer always points to the same memory location, and the data at that location can be modified through the pointer.

指针的指向固定的内存为止，但是被指向的数据可以通过指针来修改。

注意，一般来说，常指针在声明的同时就需要被初始化。但是作为函数的参数，指针随着传递给函数的同时初始化。

int* const ptr{&x}; 读作， ptr is a constant pointer to a nonconstant integer. The pointer is initialized with the address of integer variable x.

Minimum access privilege is granted.

Such a pointer always points to the same memory location, and the data at the location cannot be modified via pointer.

Pointer arithmetic is appropriate only for pointers that point to built-in array elements.

7.1. 指针的加减法

指针加减法之后的值（地址）等于运算的整数乘以该数据类型的单位内存大小。

vPtr += 2

increment： ++vPtr 和 vPtr++ 都是指向数组中的下一个元素。

decrement： --vPtr 和 vPtr-- 都是指向数组中的上一个元素。

7.2. 指针的相减

指向同一个 built-in array 的指针可以相减。

如果 vPtr 指向地址 3000，v2Ptr 指向地址 3008，那么 x = v2Ptr - vPtr 等于从 vPtr 到 v2Ptr 之间的元素个数。

比较指向不同内置数组的指针，会导致逻辑错误。

7.3. 指针赋值和比较

如果两个指针是相同类型的，可以进行赋值操作。

但是 void 是例外，任何类型的指针都可以分配给 void* 类型。

A void* pointer cannot be dereferenced.

Allowed operations only are:

  • Comparing void* pointers with other pointers
  • casting void* pointers to other pointers types
  • assigning address to void* pointers

All other operations on void* pointers are compilation errors.

可以使用 equality operator 和 relational operator 进行指针比较（指向同一个 built-in array）
