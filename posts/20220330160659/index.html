8. Function-Call Stack and Activation Records

栈的执行顺序为： last-in, first-out (LIFO) 后入先出。

通过代码示例来说明：

操作机制如下：

操作系统首先调用 main 函数，这会在栈上留下调用记录。调用记录会告诉 main 函数怎么返回操作系统，返回操作系统后的地址，以及为 main 的本地变量提供的内存。

main 函数在返回操作系统之前，调用了 square 函数，此时，栈的最顶部又加入了 square 函数的调用记录以及它的占用的内存。

当 square 函数运行结束后，它会返回到 main 函数中，也不再需要为它的本地变量提供内存，返回后 square 函数的调用在栈中就结束消失了。

最后，main 函数执行完毕，main 函数返回操作系统，它在栈中的调用记录和内存也随之结束消失，栈也就被清空了。

9. 内联函数 Inline Functions

函数调用会有一定的开销，C++提供内联函数来减少函数调用的开销。

内联函数的作用就是直接把函数的代码复制一份到需要执行的位置，从而省去了调用的开销，但是这样做也会让程序变的更加庞大，所以内联的函数一般本身比较小。实际上，现在的编译器很先进，很多编译器会自动使用内联功能来优化程序。

10. 引用

有两种给函数传递参数的方式：

  • 按值传递 pass-by-value
    • 传递的是值的一个拷贝
    • 改变这个拷贝不影响值的本身
    • 当数据很大的时候，按值传递的复制机制会导致很大的内存开销
  • 按引用传递 pass-by-reference
    • 可以直接访问、修改传递的数据
    • 有更好的运行效率和性能表现
    • 因为影响了数据本身，所以安全性较差

引用必须保证在使用的时候已经初始化，而且同一个引用名称不能再次指定给别的变量。对别名的所有操作实际上都在原数据上生效。

常引用 const Reference

比如：
void setName(std::string accountName)是按值传递，可能因为 string 对象庞大，按值传递的复制行为会拉低程序表现。这时候我们可以按引用传递。另外，这个值不需要后续修改，所以应该设置添加 const 限定符。最后的写法应该是void setName(const std::string& accountName)。

避免把引用返回到本地变量上。因为调用的函数中的引用会在函数结束时消失，所以可能会造成错误，除非要返回的变量是 static 类型。
