在前篇简单介绍过C++基础后，本篇通过一个实际的例子，简单讨论C++中的类、对象，以及字符串对象。

1. 类的基本介绍

我们使用自定义类型，即“类”，来创建一个自定义对象，即object。

我们使用#include来包含类，其中，自定义的类使用"header"，C++标准库的类使用<header>。

接口，如类的声明、类的返回类型等，都放在头文件中，函数的具体实现等都放在源文件中。这样做，可以在改变实现的时候，尽量少的改变接口类型，从而尽可能小的减少对程序整体带来的影响。

  • explicit Account(std::string accountName)

    • 构造函数有一个 string 类型的参数，叫做 accountName。
    • 当你创建一个 Account 对象的时候，必须要传递一个变量名给构造函数。这个变量名被构造函数用来初始化数据成员。
    • 因为构造函数没有任何返回值，void 类型也不行。
    • 构造函数不能被声明成 const 类型。

3. set 和 get 函数

set和get函数作为进出的唯一接口，能够有效控制对私有数据的访问和修改，这在正式的软件工程中非常有帮助。

  • set函数能够验证输入数据的正确性
  • get函数能够在获得所需数据的前提下隐藏原始的私有数据
  • 控制对私有数据的access和presentation能有效提高程序鲁棒性、安全性、可用性

5. 部分要点总结

  • 每一个创建的类都是自定的新的变量类型
  • 打包合格的类，可以被其他程序所使用
    • 为了把类给其他程序使用，将类写成头文件，从而被程序包含
    • 自定义头文件用双引号来包含，标准头文件用尖括号来包含
  • string类
    • 需要包含<string>头文件
    • 使用string类的getline()来输入含空格的语句
  • 头文件不应该使用using语句
