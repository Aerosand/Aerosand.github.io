4. Control Structures 控制结构

  • selection structure 选择结构
    • single selection statement 单选择结构
    • double selection statement 双选择结构
    • multiple selection statement 多选择结构
  • iteration structure 迭代结构

5.2. 算法细化 refinement

自上而下按步进行

以伪代码分解来说明。

  • The Top
    • Determine the class average for the quiz
  • First Refinement
    • Initialize variables
    • Input, sum and count the quiz grades
    • Calculate and print the class average
  • Second Refinement
    • Initialize variables
      • Initialize total to zero
      • Initialize counter to zero
    • Input, sum and count the quiz grades
      • Prompt the user to enter the first grade
      • Input the first grade (possibly the sentinel)
      • While the user has not yet entered the sentinel
        • Add this grade into the running total
        • Add one to the grade counter
        • Prompt the user to enter the next grade
        • Input the next grade (possibly the sentinel)
    • Calculate and print the class average
      • If the counter is not equal to zero
        • Set the average to the total divided by the counter
        • Print the average
      • else
        • Print “No grades were entered”

5.3. 程序语言实现

  • static_cast<double>(total)
    • 因为整数除法的结果肯定是除法
    • 强制类型转换成double类型
    • 但是仅转换了total的一个临时拷贝，不影响它本身
    • 这种使用 cast操作符的行为称为显式转换(explicit conversion)
  • 隐式转换(implicit conversion)
    • 上述语句的double显示转换后
    • gradeCounter在该计算中被C++编译器进行了临时隐式转换
    • 被从int类型临时提升到了double类型
    • 这也就是promotion的特性
  • setprecision()设置浮点精度
    • 参数化的流控制符
    • 必须要包含头文件<iomanip>
    • endl是非参数化的流控制符
    • 默认的浮点数是6位精度
  • fixed浮点数定点格式
    • 即使是整数类型，定点格式也会强制输出小数点和尾随零
  • rounding floating-point numbers 四舍五入
    • 数位精度设置后，C++会自动进行四舍五入

6.2. 代码实现

  • 初始化本地变量
    • 可以减少一些可能的编译错误
    • 现在更推荐不使用 = 初始化赋值语句
      • 因为等号赋值会narrow数据类型
      • int x = 12.7可以编译通过，但 x = 12
      • int x{12.7}这种写法要求前后统一，此写法就会编译报错
      • 可能避免一些隐藏的逻辑错误
